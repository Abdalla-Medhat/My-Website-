# Portfolio Prompt

> This document describes the AI-assisted workflow used to design and generate this portfolio website.

---

# Overview

This portfolio was created using a structured AI-assisted development workflow rather than a single prompt.

The objective was not only to generate code, but to produce a production-ready personal portfolio with modern UI/UX, clean architecture, responsive layouts, accessibility, and maintainable code.

The AI-generated output was then manually reviewed, refined, and optimized before deployment.

# Development Process

## 1. Prompt Engineering

The project requirements were carefully designed and organized using ChatGPT.

The prompt defined:

* Website goals
* Design language
* Visual hierarchy
* Component structure
* UI/UX guidelines
* Technology stack
* Code quality standards
* Responsiveness
* Accessibility
* Performance requirements
* Rules for handling missing information

Rather than asking the AI to simply "build a portfolio", the prompt described exactly how the final product should behave, look, and be structured.

---

## 2. Code Generation

The generated prompt was executed using:

* **OpenCode**
* **DeepSeek V4 Flash**

The AI generated the initial project structure, components, styling, animations, and application architecture.

---

## 3. Manual Review

The generated project was then manually reviewed and refined.

This included improvements such as:

* Code cleanup
* Component organization
* Better project structure
* Responsive adjustments
* UI refinements
* Performance improvements
* SEO verification
* Accessibility review

---

## 4. Repository Optimization

Before publishing the project, additional improvements were made to ensure the repository followed common GitHub best practices.

These included:

* Creating a proper `.gitignore`
* Removing AI-specific files that are not part of the application (such as `AGENTS.md` and `CLAUDE.md`)
* Improving the project documentation
* Organizing the repository structure
* Preparing the project for public release

---

# Design Philosophy

The portfolio follows a hybrid visual language.

* 70% Modern Dark UI
* 20% Soft Glassmorphism
* 10% Premium Glow Effects

The goal was to create a professional portfolio that appeals to recruiters, engineering managers, and software companies while maintaining excellent readability and usability.

---

# Technology Stack

The generated project is based on:

* Next.js
* React
* TypeScript
* Tailwind CSS
* Framer Motion
* Lucide Icons
* Modern Component Architecture

The final implementation is optimized for deployment on **Vercel**.

---

# Prompt Principles

The prompt follows several important principles.

## Single Source of Truth

All portfolio content is loaded from a single data source.

The AI should never invent information.

---

## No Fabricated Content

The model must never generate:

* fake projects
* fake experience
* fake statistics
* fake certifications
* fake companies
* fake achievements
* fake links

Missing information should always be requested before continuing.

---

## Production-Ready Code

The prompt emphasizes:

* reusable components
* semantic HTML
* responsive layouts
* accessibility
* maintainability
* clean architecture
* type safety
* SEO optimization

---

# Human Contribution

Although AI significantly accelerated development, the final result required human decisions throughout the process.

These included:

* Prompt engineering
* Design decisions
* Reviewing generated code
* Repository organization
* Documentation
* Final refinements
* Testing
* Deployment

AI served as a development assistant rather than a replacement for engineering judgment.
